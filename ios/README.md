# FlightDeck Remote (iOS)

Companion iOS app for FlightDeck — see `specs/MOBILE_REMOTE_PRD.md` at the repo
root for product scope. iOS 18+, iPhone-only in v1, SwiftUI, dark-only
"midnight mission-control" theme.

## Project generation

The `.xcodeproj` is **not committed** — it's generated from `project.yml` via
[XcodeGen](https://github.com/yonaskolb/XcodeGen). Regenerate it any time the
project structure or settings change:

```bash
brew install xcodegen   # one-time, if not already installed
cd ios
xcodegen generate
```

This produces `ios/FlightDeckRemote.xcodeproj`.

## Build

```bash
cd ios
xcodegen generate
xcodebuild build \
  -project FlightDeckRemote.xcodeproj \
  -scheme FlightDeckRemote \
  -destination 'platform=iOS Simulator,name=iPhone 16 Pro,OS=18.4' \
  -derivedDataPath .derived \
  CODE_SIGNING_ALLOWED=NO
```

## Test

```bash
ios/scripts/test.sh
```

or equivalently:

```bash
cd ios
xcodegen generate
xcodebuild test \
  -project FlightDeckRemote.xcodeproj \
  -scheme FlightDeckRemote \
  -destination 'platform=iOS Simulator,name=iPhone 16 Pro,OS=18.4' \
  -derivedDataPath .derived \
  -skipPackagePluginValidation \
  CODE_SIGNING_ALLOWED=NO
```

Both the unit test target (`FlightDeckRemoteTests`) and UI test target
(`FlightDeckRemoteUITests`) run as part of the `FlightDeckRemote` scheme.

## Structure

```
ios/
  project.yml                     XcodeGen spec (source of truth for the project)
  FlightDeckRemote/
    App/                          @main entry point + top-level router
    DesignSystem/                 Theme.swift — color tokens (PRD §11)
    Transport/                    Relay/WebSocket transport (stub)
    Features/
      Pairing/                    Pairing flow (stub)
      Projects/                   Projects list (stub)
      Sessions/                   Agent sessions list (stub)
      Chat/                       Agent chat / cleaned transcript (stub)
      Shell/                      Minimal shell terminal (stub)
      Activity/                   Activity feed (stub)
      Settings/                   Settings (stub)
    Resources/
      Assets.xcassets/            AppIcon + AccentColor placeholders
    Info.plist                    Generated by xcodegen from project.yml (not hand-edited)
  FlightDeckRemoteTests/           Unit tests (Swift Testing)
  FlightDeckRemoteUITests/         UI tests (XCTest / XCUIApplication)
  scripts/test.sh                 xcodegen + xcodebuild test, one command
```

Each `Features/*` folder currently holds a single placeholder view wired up
just enough to compile and be navigable; feature teams fill these in.

## Notable decisions / deviations

- **Swift language mode:** Swift 5 mode (`SWIFT_VERSION: 5.0`), not Swift 6
  strict concurrency. This keeps the scaffold friction-free for feature teams
  starting out; revisit once the Transport layer (actors, `Sendable` models)
  is designed in earnest.
- **SwiftTerm dependency:** added via Swift Package Manager
  (`https://github.com/migueldeicaza/SwiftTerm`, from `1.14.0`), linked to the
  app target but not yet imported/used anywhere — the Shell feature will
  consume it.
- **`.xcodeproj` (and the app target's generated `Info.plist`) are
  gitignored.** `project.yml` is the source of truth for both; run
  `xcodegen generate` after cloning or pulling changes to `project.yml`.
- **No code signing configured.** `CODE_SIGNING_ALLOWED=NO` / automatic
  signing with no team — sufficient for simulator builds; a real team/profile
  will be needed for device builds and TestFlight later.
- **Dark-only:** `UIUserInterfaceStyle` is forced to `Dark` in the generated
  Info.plist, and the root view also sets `.preferredColorScheme(.dark)`
  in-process.
